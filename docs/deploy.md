# Инструкция по развёртыванию приложения
## Быстрый старт
1. Создайте любую директорию и поместите техническую литературу (документы в формате PDF, DOCX, TXT), с которой работает бот. Желательно, чтобы полный путь до этой директории не содержал пробелов. 
2. Создайте любую директорию. Можно создать в ней пустой файл `database.db`, но это необязательно (в таком случае он будет создан автоматически). Желательно, чтобы полный путь до этой директории не содержал пробелов.
3. Создайте Telegram-бота через BotFather (например, с помощью [этой инструкции](https://core.telegram.org/bots/tutorial)). Скопируйте токен бота.
4. Установите Docker и Docker Compose на целевой ПК или сервер.
5. Скопируйте файл `env_example.txt`, переименуйте копию в `.env` и также поместите в корневую директорию.
6. Отредактируйте файл `.env`:
  - Для переменной `LSB_BOT_TOKEN` укажите токен бота, полученный через BotFather.
  - Для переменной `LSB_ADMINS_ON_START` желательно указать Telegram ID (уникальный числовой идентификатор) пользователей, которые будут администраторами бота.
    - Их нужно указывать через запятую без пробелов
    - Можете не указывать ничего, тогда при первом старте бота администратору нужно будет вызвать команду `/whoami` - бот выведет ваш Telegram ID. Потом можно добавить его в переменную и перезапустить бота.
  - Не нужно менять переменные `LSB_DB_PATH` и `LSB_LITERATURE_PATH`, т. к. они указывают на пути внутри контейнера.
7. Отредактируйте файл `docker-compose.yml`:
  - В разделе `volumes` укажите:
```yaml
volumes:
  - ./.env:/app/.env
  - <path_to_data>:/app/data
  - <path_to_literature>:/app/data/literature:ro
```
  - Где `<path_to_data>` - путь до директории с базой данных, `<path_to_literature>` - путь до директории с литературой.
  - Даже на Windows не рекомендуется использовать обратные слэши `\` в путях, а также двоеточия `:` для буквы диска (потому что это может быть ошибочно воспринято как разделитель).
    - Например, если путь до директории с литературой:
      - `C:\Users\somebody\Documents\Literature`
    - То нужно указать путь как:
      - `/C/Users/somebody/Documents/Literature`
8. Откройте терминал, командную строку или PowerShell.
9. Перейдите в корневую директорию бота.
10. Сборка:
```bash
docker compose build
```
11. Запуск:
```bash
docker compose up -d
```
12. Проверка логов:
```bash
docker compose logs -f
```
13. Выключение:
```bash
docker compose down
```
